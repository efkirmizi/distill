Project Description

Compressing deep networks for the edge — without breaking them

Modern vision models carry millions of parameters and billions of operations per inference, far beyond the memory and power budgets of drones, IoT sensors and medical-imaging hardware. This project resolves the two failure modes of existing compression by unifying them into one joint optimization.

Tensor decomposition breaks

Factorizing a pre-trained network and fine-tuning it sequentially triggers vanishing gradients and catastrophic accuracy loss at high compression — the layers become structurally fragile.

Distillation collapses

Knowledge distillation recovers accuracy but breaks down under the capacity gap at extreme compression — and becomes ill-defined once a student's layers are tensor-factorized.

Our joint framework

Build the compressed students from scratch and co-train them under one frozen teacher in a single pass — compression and knowledge transfer happen together, not in fragile stages.

How it works

A three-stage automatic pipeline

Every design choice is derived from the teacher itself — no global compression ratio, no validation search, and no hand-tuned loss coefficients.

Stage 1

PURSUhInT

Where to supervise. Clusters the teacher's layers by representational similarity (CCA / CKA) and selects one non-redundant hint point per cluster — K=3 on CIFAR, K=4 on ImageNet.

Stage 2

EVBMF ranks

How much to compress. Reads the teacher's trained weight spectrum and sets each layer's rank automatically via a Marchenko–Pastur noise threshold — layer-adaptive, fully automatic.

Stage 3

Dual co-training

Train together. A CP and a Tucker student are co-trained under one shared teacher forward pass, balancing cross-entropy, KD and attention transfer with learned uncertainty weights.

Technical details

Two complementary decompositions, one teacher

Both students share the same base architecture but factorize every eligible convolution two different ways, giving a controllable trade-off between size and accuracy.

CP decomposition

4 ops / conv
𝒲 ≈ Σ aᵣ ⊗ bᵣ ⊗ cᵣ ⊗ dᵣ

Factorizes each convolution into four separable operations (1×1 → depthwise kH → depthwise kW → 1×1). Separable spatial filtering yields the highest compression at a slightly larger accuracy cost.

Params: R(Cin+kH+kW+Cout) Tightest budget

Tucker decomposition

3 ops / conv
𝒲 ≈ 𝒢 ×₁ Uout ×₂ Uin

Keeps a dense kH×kW core convolution (1×1 → core → 1×1) that preserves cross-channel correlations — delivering higher accuracy and lower latency despite more parameters than CP.

Dense core Best accuracy / latency

Automatic loss balancing

Kendall uncertainty
ℒ = Σi ∈ {CE, KL, AT} ( e−sᵢ ℒᵢ + sᵢ )

The cross-entropy, knowledge-distillation and attention-transfer terms operate at wildly different numerical scales. Learnable log-variance weights si equalize their gradient contributions to within 10% automatically — eliminating any manual coefficient search.

Real measured speedups

Single-image CPU inference on the ResNet-34 → ResNet-18 pair falls from 265.98 ms (teacher) to 135.88 ms (CP) and 125.91 ms (Tucker). Tucker runs faster despite carrying 2.4× more parameters — its 3-operation kernel chain issues fewer sequential launches than CP's 4-op chain.
